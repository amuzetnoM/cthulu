# Release entry template

Use this template for new releases. Keep entries concise and use the groups below:

## [x.y.z] - YYYY-MM-DD

### Added
- Feature summary (file path(s) optional)

### Changed
- Configuration or behavior changes

### Deprecated
- Deprecated features and migration notes

### Removed
- Removed features and rationale

### Fixed
- Bug fixes with brief context

### Security
- Security updates or advisories

---

Notes:
- Keep entries one sentence each when possible.
- Link to related PRs or issues in parentheses, if helpful: (PR #123)
