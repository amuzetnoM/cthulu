Herald v4.0.0 — MAJOR RELEASE

Summary
**MAJOR RELEASE**: Multi-Strategy Trading System with Dynamic Selection and Next-Generation Indicators

Added
- Multi-Strategy Framework
  - 4 New Advanced Trading Strategies: EMA Crossover, Momentum Breakout, Scalping, Enhanced SMA Crossover
  - `strategy/strategy_selector.py`: Autonomous strategy switching engine with market regime detection (5 regimes) and weighted scoring
- Next-Generation Indicators
  - `indicators/supertrend.py`, `indicators/vwap.py`, Anchored VWAP, Enhanced multi-period RSI
- Data Layer & Infrastructure
  - `data/layer.py`: Unified OHLCV processing and strategy-specific data preparation
- Enhanced GUI
  - `ui/desktop.py`: Tkinter-based monitoring interface with strategy display, metrics and manual trade UI
- Configuration & Modes
  - `config_ultra_aggressive.json` for ultra-aggressive production-ready configuration
- Metrics & Observability
  - Improved `observability/metrics.py` with risk:reward tracking, expectancy, rolling Sharpe, per-symbol aggregates

Changed
- Version bumped (3.3.1 → 4.0.0) and main orchestrator (`__main__.py`) integrated with StrategySelector and GUI auto-launch
- Performance: improved polling and execution performance; tighter risk controls and updated indicator loading

Testing
- 27+ comprehensive unit tests covering advanced strategies and next-gen indicators

Documentation
- `UPGRADE_GUIDE.md` and `IMPLEMENTATION_SUMMARY.md` added/updated
- README updated with v4.0.0 highlights

Breaking Changes
- Major version bump (3.x → 4.0); architecture and configuration extended to support multi-strategy operation

Migration Path
1. Test in dry-run mode with individual strategies
2. Enable dynamic strategy selection
3. Integrate GUI monitoring
4. Deploy with ultra-aggressive configuration (if desired)

Notes
- No new third-party dependencies beyond those already in requirements
- See `docs/changelog/CHANGELOG.md` and `UPGRADE_GUIDE.md` for detailed migration instructions
