Data Flow Pipeline

Market Data → Data Layer → Strategy → Signal → Risk Check → Execution → Persistence → Monitoring ┌─────────────────┐ │ Market Data │ ← MetaTrader 5 Terminal └────────┬────────┘ │ ▼ ┌─────────────────┐ │ Data Layer │ ← Normalization, Indicators, Caching └────────┬────────┘ │ ▼ ┌─────────────────┐ │ Strategy │ ← Signal Generation (SMA, etc.) └────────┬────────┘ │ ▼ ┌─────────────────┐ │ Signal │ ← Standardized Signal Dataclass └────────┬────────┘ │ ▼ ┌─────────────────┐ │ Risk Manager │ ← Position Sizing, Limit Checks └────────┬────────┘ │ ▼ ┌─────────────────┐ │ Execution │ ← Idempotent Order Submission └────────┬────────┘ │ ▼ ┌─────────────────┐ │ Persistence │ ← Database Storage └────────┬────────┘ │ ▼ ┌─────────────────┐ │ Observability │ ← Logging, Metrics, Alerts └─────────────────┘

herald/connector/

MT5Connector: Manages MetaTrader 5 terminal connection with automatic reconnection, rate limiting, and health checks. Provides thread-safe access to market data and order execution.

herald/data/

DataLayer: Normalizes OHLCV data from MT5 into standardized pandas DataFrames. Calculates technical indicators (SMA, EMA, ATR, RSI) and handles multi-timeframe resampling.

herald/strategy/

Strategy (ABC): Abstract base class defining the strategy interface. Implementations like SmaCrossover generate Signal objects with entry/exit criteria and metadata.

herald/execution/

ExecutionEngine: Handles order submission with idempotency guarantees via client_tag tracking. Manages partial fills, position closure, and order status reconciliation.

herald/risk/

RiskManager: Approves or rejects signals based on position sizing rules, exposure limits, and daily loss thresholds. Provides emergency shutdown capability.

herald/persistence/

Database (Planned): SQLAlchemy-based ORM for storing signals, orders, trades, positions, and metrics. Supports historical analysis and performance tracking.

Quick Start in 3 Steps

Get Herald running on your system in minutes with our automated setup scripts.

Step 1: Automated Setup Windows 
# Clone or download Herald
cd C:\workspace\Herald

# Run automated setup (PowerShell)
.\scripts\setup.ps1

# The script will:
# ✓ Check Python 3.10+
# ✓ Create virtual environment
# ✓ Install dependencies
# ✓ Initialize database
# ✓ Copy config template
Step 2: Configure MT5 YAML 
# Edit config/config.yaml
mt5:
  login: 12345678
  password: "your_password"
  server: "Broker-Demo"

risk:
  max_position_size_pct: 0.02  # 2% per trade
  max_daily_loss_pct: 0.05     # 5% daily limit
  max_total_positions: 3

strategy:
  name: "sma_crossover"
  short_window: 20
  long_window: 50
Step 3: Run Herald Python 
# Activate virtual environment
.\venv\Scripts\Activate.ps1

# Run in dry-run mode first
python -m herald --dry-run

# Run with live trading
python -m herald

# Run with custom config
python -m herald --config custom.yaml

# Check system health
python -m herald --health-check

Example Strategy Usage Python 
from herald.connector import MT5Connector
from herald.data import DataLayer
from herald.strategy import SmaCrossover
from herald.risk import RiskManager
from herald.execution import ExecutionEngine

# Initialize components
connector = MT5Connector(config)
data_layer = DataLayer(connector)
strategy = SmaCrossover(config)
risk_mgr = RiskManager(config, connector)
executor = ExecutionEngine(connector)

# Main trading loop
while True:
    # Get market data
    df = data_layer.get_ohlcv("XAUUSD", "M15")
    
    # Generate signal
    signal = strategy.on_bar(df)
    
    if signal:
        # Risk approval
        if risk_mgr.approve(signal):
            # Execute trade
            result = executor.submit_order(signal)
            console.log("Order executed: " + result);
Database Schema SQL 
-- Signals table
CREATE TABLE signals (
    id INTEGER PRIMARY KEY,
    timestamp TEXT,
    symbol TEXT,
    signal_type TEXT,
    entry_price REAL,
    stop_loss REAL,
    take_profit REAL
);

-- Orders table
CREATE TABLE orders (
    id INTEGER PRIMARY KEY,
    signal_id INTEGER,
    order_type TEXT,
    status TEXT,
    filled_price REAL,
    FOREIGN KEY (signal_id) 
        REFERENCES signals(id)
);

-- Trades table with P&L
CREATE TABLE trades (
    id INTEGER PRIMARY KEY,
    order_id INTEGER,
    entry_time TEXT,
    exit_time TEXT,
    profit_loss REAL
);
