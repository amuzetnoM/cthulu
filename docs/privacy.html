Privacy Policy

Last updated: December 28, 2025

Overview

Cthulu is an open-source autonomous trading system that runs locally on your machine. We are committed to protecting your privacy and being transparent about data handling.

Data Collection

Cthulu does not collect, transmit, or store any personal data on external servers. All data remains on your local machine:

  • Trading Data: All trading activity, positions, and performance metrics are stored in your local SQLite database (Cthulu.db)
  • Log Files: System logs are stored locally in the logs/ directory
  • Configuration: Your trading configuration, API keys, and credentials are stored locally and never transmitted
  • Market Data: Market data is fetched from your broker (MetaTrader 5) and stored in local cache

Third-Party Services

Cthulu may interact with the following third-party services:

  • MetaTrader 5: Direct connection to your MT5 terminal for order execution and market data
  • News Providers: Optional integration with FRED and Trading Economics APIs for economic news (requires your API keys)
  • Prometheus/Grafana: Optional local monitoring stack (all data stays on your network)

Data Security

Security best practices:

  • Use environment variables or .env files for sensitive credentials
  • Never commit API keys or passwords to version control
  • Restrict file permissions on config files and databases
  • Use encrypted connections when available
  • Regularly review your MT5 account access logs

Open Source

Cthulu is open-source software. You can review the entire codebase to verify how data is handled. The source code is available at: github.com/amuzetnoM/herald

Your Rights

Since all data is stored locally on your machine, you have complete control:

  • Delete any or all data at any time by removing database files or log files
  • Modify the source code to customize data handling
  • Export your data in any format you choose
  • Control what data is collected through configuration

Contact

For questions or concerns about privacy, please open an issue on GitHub or contact the project maintainers.
